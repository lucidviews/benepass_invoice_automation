# Invoice Automation — Robustness Improvements Design

**Date:** 2026-03-27
**Scope:** Run logging, macOS notifications, screenshot-on-failure debugging, Playwright MCP integration in SKILL.md, Cloudflare handling

---

## Problem Statement

Two pain points with the current setup:

1. **Adding platforms is tedious** — specifically, debugging selector issues after a downloader is generated requires manual inspection with no automated help.
2. **No confidence after a run completes** — there is no summary of what was submitted or any persistent record of the run.

Additionally, Cloudflare's "Confirm you are human" challenge blocks some platforms. The project already has `playwright-extra` and `puppeteer-extra-plugin-stealth` installed but they are not wired up.

---

## Design

### 1. Run Logging + macOS Notification

**New file: `logger.js`**

A small module with two exports:

- `log(message)` — writes timestamped output to both `console.log` and a log file at `~/Documents/invoices/YYYY-MM/run.log`
- `notifyRunComplete(results)` — fires a macOS notification via `osascript` with a one-line summary

Notification format:
- Full run: `"3 submitted: Vodafone €46.11, Urban Sports €29.00, Freenet €19.99"`
- Dry run: `"Dry run complete — 3 platforms checked"`
- On error: `"Automation failed: <error message>"`

**Changes to `index.js`:**
- Replace `console.log` calls with `logger.log`
- Call `logger.notifyRunComplete(results)` at the end of a successful run and in the dry-run branch
- Pass error message to notification in the `.catch` handler

### 2. Screenshot on Failure + Playwright MCP Debugging

**Changes to `downloader-patterns.md` Strategy A:**

Update the `try/finally` block to catch errors and save a screenshot before rethrowing:

```js
} catch (err) {
  await page.screenshot({ path: `/tmp/<platform>-error.png` });
  throw new Error(`[MyPlatform] Failed at step X: ${err.message} — screenshot saved to /tmp/<platform>-error.png`);
}
```

Every downloader generated by the skill includes this pattern. The screenshot path is included in the thrown error message so it surfaces in the run log.

**Changes to `SKILL.md` Step 5:**

When a dry run fails, the debugging flow becomes:

1. Note the screenshot path from the error message
2. Use the Playwright MCP (`browser_navigate`, `browser_snapshot`, `browser_take_screenshot`) to open the platform's invoice page live
3. Inspect the DOM snapshot to identify the correct selector
4. Update the downloader and re-run

The updated Step 5 text explicitly instructs Claude to use the Playwright MCP for live inspection rather than asking the user to debug manually.

### 3. Cloudflare Handling

**Changes to `downloader-patterns.md`:**

**Strategy A upgrade — stealth plugin (default):**

Replace raw `chromium` with the stealth-wrapped version. The project already has the required packages.

```js
const { chromium } = require('playwright-extra');
const StealthPlugin = require('puppeteer-extra-plugin-stealth');
chromium.use(StealthPlugin());
```

This is a one-import change. All other Strategy A code remains identical. Prevents Cloudflare from triggering in the first place for most sites.

**New Strategy C — Manual Cloudflare pause:**

For sites where stealth is not enough. After `page.goto`, detect the challenge:

```js
if (page.url().includes('challenges.cloudflare.com') ||
    await page.locator('iframe[src*="challenges.cloudflare.com"]').count() > 0) {
  notify('Invoice Automation', `Cloudflare challenge on [Platform] — solve it in the browser, then press Enter`);
  console.log('[Platform] Waiting for you to solve the Cloudflare challenge...');
  await keypress();
}

function keypress() {
  return new Promise(resolve => process.stdin.once('data', resolve));
}
```

No paid services, no proxy dependencies. Reliable for a once-a-month personal tool.

**Changes to `SKILL.md` Step 2 (gather platform details):**

Add question 4a (between current questions 4 and 5):

> **Cloudflare challenge**: Does the site show a "Confirm you are human" Cloudflare challenge when you visit it? If yes → Strategy C wrapper will be added to the downloader.

**Precedence of strategies:**

| Situation | Strategy |
|---|---|
| Normal site | A (stealth) |
| Cloudflare challenge | A (stealth) + C wrapper |
| Deeply hostile / real cookies needed | B (real browser) |
