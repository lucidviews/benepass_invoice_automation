# Benepass Invoice Automation

Automatically download invoices from your subscription services and submit reimbursements to Benepass each month.

## How it works

1. Downloads invoices from configured platforms (via browser automation)
2. Extracts the amount from each PDF
3. Applies your Benepass stipend caps
4. Submits each expense to Benepass (with Okta SSO authentication)

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Configure your environment

```bash
cp .env.example .env
```

Open `.env` and fill in your Benepass email and any credentials for the platforms you add.

### 3. Add your platforms with Claude

Open a Claude Code session in this directory. Claude will automatically offer to walk you through setup — or just describe what you need:

> "I need to automate invoices from Spotify and my gym"

Claude will ask you a few questions per platform and generate the downloader code and configuration automatically.

### 4. Run

```bash
# Dry run — downloads invoices and shows what would be submitted, without touching Benepass
node index.js --dry-run

# Submit everything
node index.js

# Target a specific platform only
node index.js --dry-run --only=spotify
node index.js --only=spotify,gym
```

## First run

On the first real run, Benepass will require Okta authentication. A macOS notification will appear — touch your fingerprint in Okta Verify when prompted. The session is saved so you won't need to do this every time.

## Project structure

```
├── index.js              # Main orchestrator — add your platforms here
├── benepass.js           # Benepass submission (handles Okta SSO automatically)
├── pdfParser.js          # Extracts amounts from invoice PDFs
├── downloaders/          # One file per platform, generated by Claude
├── .env                  # Your credentials (not committed)
└── .env.example          # Template for required env vars
```

## Stipend caps

If your company caps reimbursements per category, add them to the `CAPS` object in `index.js`. If two platforms share a combined cap (e.g. two phone bills under one internet stipend), give them the same `category` and `cap` value — the script will split proportionally.
